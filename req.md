# 需求文档

## 概览

- 核心需求：这是一个项目合同统计提醒系统，用于自动计算各个项目的应收、应付款，并在每个付款周期进行提醒和统计。
- 语言：java
- 平台：在 macOS 进行开发和测试，在 Windows 平台进行部署（部署尽可能简单，不要有复杂的依赖工具安装）
- 运行方式：web 应用程序，服务端在本地后台运行，前端在浏览器中打开
- 风格：轻量化，UI 简洁美观，有高级感

## 功能：用户录入合同

程序的输入只有一个方式：用户录入一份合同，录入时用户需填写以下内容：

- 首选项：应收款 or 应付款，对应了在该合同中，用户是甲方还是乙方
- 关键词集合：签约单位，对方单位，项目名称，签约时间，签约面积，起租时间，免租期时间，合同终止时间，月租金，月物业费，缴费周期(单位：月)

备注：所有日期相关的关键词，在输入时提供 web 应用常见的日期选择器。其余关键词直接让用户输入字符串即可。

## 功能：对每份合同，统计并显示以下指标

- 每个缴费周期应[收，付]款：（月租金 + 月物业费）*（缴费周期）
- 合同期应[收，付]款：每个缴费周期应[收，付]款 *（合同终止时间-起租时间-免租期）

## 功能：收付款提醒

- 免租期内不考虑，仅在合同期内进行提醒。
- 每个缴费周期前十五天生成提醒选项卡
    - 完成后用户可关闭提醒
    - 若未完成，用户可选择将该缴费事项加入欠费清单

## 程序界面

分为左中右三栏

- 左侧栏

提醒栏，从上到下依次排列所有提醒选项卡，每个选项卡包含以下信息：签约单位、项目名称、当前周期应[收，付]款，当前周期开始结束时间，底部显示一个进度条，离缴费日期越近越短

- 中间区域

从上到下，依次排列所有合同，每个合同包含所有输入的详细信息以及统计指标

- 右侧区域

欠费清单，参考提醒栏，没有进度条

## 第一次追加需求

- 所有付款合同和收款合同的选项卡用不同的颜色加以区分
- 中间区域，每个合同添加修改删除按钮
- 合同输入框记忆复选（比如我输入过一个项目名称，下次输入时可以直接选择历史输入过的项目名称）
- 左侧提醒栏按照收付分开，加一个下拉框，只保留一个绿色的完成按钮
- 右侧栏改为合同统计栏：每个合同一个选项卡，包含所有合同，每个选项卡内容如下：（下面以收款为例，付款同理）
    - 本期应收款
    - 累计应收款
    - 累计实收款
    - 和左侧提醒栏一样的进度条
    - 每期可手动输入本期实收款，用以统计‘累计实收款’，用户可以选择‘全额收款’或是在非全额收款的情况下，手动输入实收款，完成后进入下一个缴费周期
- 左侧和右侧栏的标题横着写，不要竖着
- 中间区域最上方的标题删去
- 中间区域，输入合同不要默认显示在首页，做成一个通过按钮触发的二级菜单，用户一点开程序看到的应该是以后合同的详情

## 第二次追加需求

- 右侧栏：另外统计一个数据点：欠费统计，也就是应收和实收之间的差距；另外，仅在下一个周期开始前 15 天，才将下一周期的款项纳入应收款中
- 右侧栏：‘当前周期’放到选项卡上方（项目名称下方）
- 右侧栏：实收金额输入框内，删掉框内的文本，以及框右侧的的上下按钮，把框做大一些，不要太小

删掉文本：按合同查看本期、累计与实收实付统计，并推进到下一期。
删掉文本：字符串输入框支持历史值快速选择
删掉文本：合同修改成功
删掉文本：提醒按收付区分展示，仅保留完成按钮。
删掉文本：录入合同功能区内，所有时间相关输入框内不要有 yy/mm/日 这样的文本，只保留一个日期选择器按钮即可
删掉：所有选项卡右下角的小圆形图案
删掉：右侧栏选项卡中的‘所有周期已完成’

- 中间区域：合同列表底部，合同总金额等统计项中，数值不要靠右，要靠着数值对应的字段名，比如类似：合同期总金额：xxxx
- 中间区域：录入和合同列表中间插入一个功能区，叫做利润统计：以签约单位+项目名称作为聚合标准，统计名下的所有合同的累计应收、实际收款以及差额（也就是该项目的利润）

- 左侧区域：下拉框放在‘提醒栏’标题下方

最后，做一些你能想到的 UI 美化，比如给删除按钮变成红底，把各个按钮做成类似的常见样式，比如绿色的完成按钮，红色的删除按钮，黄色的修改按钮等
